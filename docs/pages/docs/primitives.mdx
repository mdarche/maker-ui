# Primitives

Maker UI Primitives are basic JSX elements that support `css` and `breakpoints` props.

You can use these components just as you would their normal HTML counterparts. They also forward any external refs that you might need to assign for measurements or layout effects.

Just import the capital letter version of your desired JSX element or use an alias:

## Default Breakpoints

If you don't use a Maker UI layout, the default breakpoints are `[768, 960, 1440]` (in pixels).

If you are using the full layout system, all nested primitive components will abide by the global breakpoints that you declare in your options configuration:

```jsx
// Example Layout component file
import {
  Layout,
  ...
} from 'maker-ui`

const options: MakerOptions = { breakpoints: [450, 600, 960] }

const MyLayout = () => (
  <Layout options={options}>
    ...
  </Layout>
)

// Page or child component

export default HomePage = () => (
  <Div css={{ color: ['blue', 'red', 'green'] }}>
    <h1>Home page</h1>
  </Div>
)

```

In this example, the heading color would correspond with **min-width** `450px`, `600px`, and `960px` media queries.

You are free to use any unit for your breakpoints array including `vw`, `em`, `pt`, etc. If you use a number type instead of a full string, your value will be converted into pixels.

### Div

Supports all props of the `<div>` tag.

This is most likely the primitive you need for the job. If you use it as the outermost wrapper of your component, you can set all nested, responsive styles. See [Nesting Selectors](/docs/styling-components/#nesting-selectors) for details.

```jsx
import { Div } from 'maker-ui'
```

### Flex

Flex is a `<Div>` component with a predefined flex display and a few handy shortcuts.

```jsx
import { Flex } from 'maker-ui'
```

You can use the following custom props instead of adding your flex settings to the `css` prop:

|  Prop   | Description                                                                                                                        |
| :-----: | ---------------------------------------------------------------------------------------------------------------------------------- |
|  align  | <div className="prop">string &#124; string[]</div> A shortcut for the responsive **align-items** CSS property.                     |
| column  | <div className="prop">boolean</div> When true, this will set **flex-direction** to **column**. This value is **false** by default. |
|  flex   | <div className="prop">string &#124; string[]</div> A shortcut for the responsive **flex** CSS property.                            |
| inline  | <div className="prop">boolean</div> When true, this will set **display** to **inline-flex**.                                       |
| justify | <div className="prop">string &#124; string[]</div> A shortcut for the responsive **justify-content** CSS property.                 |
|  wrap   | <div className="prop">boolean</div> When true, this will set **flex-wrap** to **wrap**.                                            |

### Grid

Grid is a `<Div>` component with a predefined CSS Grid display and a few handy shortcuts.

```jsx
import { Grid } from 'maker-ui'
```

You can use the following custom props instead of adding your grid settings to the `css` prop:

|  Prop   | Description                                                                                                                                                                           |
| :-----: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
|  areas  | <div className="prop">string &#124; string[]</div> A shortcut for the responsive **grid-template-areas** property.                                                                    |
| center  | <div className="prop">boolean</div> When true, this will vertically and horizontally center all content in each direct descendent container by setting **place-items** to **center**. |
| columns | <div className="prop">string &#124; string[]</div> A shortcut for the responsive **grid-template-columns** property.                                                                  |
|   gap   | <div className="prop">number &#124; string &#124; (number &#124; string)[] </div> A shortcut for the responsive **gap** property.                                                     |
|  rows   | <div className="prop">string &#124; string[]</div> A shortcut for the responsive **grid-template-row** property.                                                                      |

## JSX

If you don't want to import primitives or if you would rather use a JSX runtime that automatically parses elements for `css` and `breakpoints`, you can use Maker UI's JSX pragma instead.

```jsx
/** @jsx jsx */
import { jsx } from 'maker-ui`

// This will work the same as importing the <Div> and <H1> primitives

export const MyComponent = () => (
  <div
    breakpoints={[500, 960]}
    css={{ padding: [20, 30, 40] }}
  >
    <h1 css={{ color: ['red', 'blue'] }}></h1>
  </div>
)
```

Remember to add the comment from the above example to the top of your file so
you don't accidentally use the default React JSX runtime.
