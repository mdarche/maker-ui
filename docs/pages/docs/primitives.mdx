# Primitives

Maker UI Primitives are `css` and `breakpoints` enabled React elements that help you to quickly prototype your ideas and build responsive designs.

You can use these components just as you would their normal HTML counterparts. You can import the capital letter version of the normal JSX element or use an alias if applicable:

## Default Breakpoints

If you are using the full Maker UI layout system, all nested primitive components will use the global breakpoints you specify in your options configuration:

```jsx
// Example Layout component file
import {
  Layout,
  ...
} from 'maker-ui`

const options: MakerOptions = { breakpoints: [768, 960, 1200] }

const MyLayout = () => (
  <Layout options={options}>
    ...
  </Layout>
)

// Page or child component

export default HomePage = () => (
  <Div css={{ color: ['blue', 'red', 'green' ]}}>
    <h1>Home page</h1>
  </Div>
)

```

In the above example the heading tag color styles would correspond with **min-width** `768px`, `900px`, and `1200px` media queries.

### Div

Supports all props of the `<div>` tag.

This is most likely the primitive you need for the job. If you use it as the outermost wrapper of your component, you can set all nested, responsive styles. See [Nesting Selectors](/docs/styling-components/#nesting-selectors) for details.

```jsx
import { Div } from 'maker-ui'
```
