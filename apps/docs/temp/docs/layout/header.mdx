The Header component renders a `<header>` tag that contains all components for desktop and mobile navigation.

## Props

|      Prop      | Description                                                                                                                                                                                                                    |
| :------------: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
|    absolute    | <Type name="boolean" /> Overrides `header.absolute` from Maker UI options.                                                                                                                                                     |
|   background   | <Type name="ResponsiveString" /> Overrides the Header's default `--color-bg_header` CSS variable that you can set in Maker UI options. **Note:** This doesn't have to be a color - you can use a gradient or an image as well. |
|      css       | <Type name="object" /> Applies responsive CSS styles to the component.                                                                                                                                                         |
|     sticky     | <Type name="boolean"/> Overrides `header.sticky` from Maker UI options.                                                                                                                                                        |
| stickyOnMobile | <Type name="boolean"/> Overrides `header.stickyOnMobile` from Maker UI options.                                                                                                                                                |
| stickyUpScroll | <Type name="boolean"/> Overrides `header.stickyUpScroll` from Maker UI options.                                                                                                                                                |

<Callout type="suggestion">
  The override props are designed for conditional use cases or runtime logic.
  Whenever possible, use Maker UI options instead.
</Callout>

## Maker UI Options

Your options configuration has a `header` object with the following properties:

|         Option          | Description                                                                                                                                                                                                                                                                                                                                                                                            |
| :---------------------: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
|        absolute         | <Type name="boolean" /> If true, the header will be absolutely positioned so site content begins at the top of the viewport. <DefaultValue value="false" />                                                                                                                                                                                                                                            |
|       breakpoint        | <Type name="string-number" /> A breakpoint that controls when the header switches from desktop to mobile navigation. You may also use an index to access a breakpoint in the `options.breakpoints` array. <DefaultValue value="0"  note="The first index of options.breakpoints" />                                                                                                                    |
|       colorButton       | <Type multi={["'default'", 'React.ReactNode', '((currentMode?: string, attributes?: object) => React.ReactNode)' ]} /> Helps you customize the header's color toggle. By default, it prints the name of the current color mode, but you can use a custom React component or a JSX callback to animate the icon's state.                                                                                |
|        dropdown         | <Type nested={{ caret: ['boolean', 'React.ReactElement'], transition: ["'scale'", "'fade'", "'fade-down'", "'fade-up'", "'none'"] }}></Type> Controls the header's default dropdown menu settings. Yo u can set a custom caret component to replace the default down caret. You can also choose a dropdown animation.                                                                                  |
|      errorBoundary      | <Type name="boolean" />When true, this will wrap the inner contents of the Topbar in an Error Boundary that you can customize with `options.errors`.                                                                                                                                                                                                                                                   |
|        maxWidth         | <Type name="ResponsiveScale" /> The max-width of the header's content container. Can be a responsive array. <DefaultValue value="1460" />                                                                                                                                                                                                                                                              |
|       menuButton        | <Type multi={["'default'", 'React.ReactNode', '((isOpen?: boolean, attributes?: object) => React.ReactNode)' ]} /> Lets you customize the mobile menu toggle. This will render a traditional hamburger icon by default, but you can use a custom React component or a JSX callback to animate the icon's state.                                                                                        |
|      menuOverflow       | <Type>'wrap' &#124; 'scroll'</Type> Lets you decide if your nav menu should wrap to the next line or scroll horizontally when the menu is larger than its container width.                                                                                                                                                                                                                             |
|      mobileNavType      | <Type multi={["'basic'", "'basic-menu-left'", "'logo-center'", "'logo-center-alt'" ]} flex /> The Navbar's layout on mobile devices. [See below](#nav-types) for details. <DefaultValue value="basic" />                                                                                                                                                                                               |
|         navType         | <Type multi={["'basic'", "'basic-left'", "'basic-right'", "'center'", "'split'", "'minimal'", "'minimal-left'","'minimal-center'","'reverse'", ]} flex /> The Navbar's layout on desktop devices. [See below](#nav-types) for details. <DefaultValue value="basic" />                                                                                                                                  |
|       scrollClass       | <Type nested={{ scrollTop: ['number'], className: ['string'] }}></Type> Adds a custom class to the header when the user has scrolled past a specified point.                                                                                                                                                                                                                                           |
|     showColorButton     | <Type value="boolean" /> A boolean that determines if the header nav area should display a color toggle button. <DefaultValue value="true" />                                                                                                                                                                                                                                                          |
| showColorButtonOnMobile | A boolean that lets you hide the header's color toggle button on mobile. <DefaultValue value="true" />                                                                                                                                                                                                                                                                                                 |
|   showWidgetsOnMobile   | <Type value="boolean" /> A boolean that lets you hide the nav widget area slot on mobile.<DefaultValue value="false" />                                                                                                                                                                                                                                                                                |
|         sticky          | <Type value="boolean" /> A boolean that determines if the header sticks to the top of the viewport while scrolling. <DefaultValue value="false" />                                                                                                                                                                                                                                                     |
|     stickyOnMobile      | <Type value="boolean" /> A boolean that determines if the header sticks to the top of the viewport while scrolling on mobile. <DefaultValue value="false" />                                                                                                                                                                                                                                           |
|     stickyUpScroll      | <Type multi={["'boolean'", "{ delay: number, start: number }"]} /> Lets you hide the header on down scroll and reveal it when scrolling back up. You can use the `delay` property to set a time in milliseconds after the upscroll event to begin the reveal transition. You can also set a `start` scrollTop value (pixels) to determine when the effect should begin. <DefaultValue value="false" /> |

## Usage

Add `Header` to the beginning of your layout tree and wrap it around `Navbar` and `MobileMenu`:

```jsx {15-18}
import {
  Layout,
  Header,
  Navbar,
  MobileMenu
  Content,
  Main,
  Footer
} from 'maker-ui'

// This example omits key props like menus, options, etc.

export const MyLayout = ({ children }) => (
  <Layout>
    <Header>
      <Navbar />
      <MobileMenu />
    </Header>
    <Content>
      <Main>{children}</Main>
    </Content>
    <Footer />
  </Layout>
)
```
